## Logic

The key insight is to combine:
- **Exercise 1.16**: The invariant technique - keep an accumulator `acc` such that `acc + a * b` remains constant throughout
- **Exercise 1.17**: The Russian peasant multiplication algorithm using double/halve

**Invariant**: `acc + a * b = original_a * original_b`

**State transformations**:
- If `b = 0`: answer is `acc`
- If `b` is even: `acc + a * b = acc + (2a) * (b/2)` → double `a`, halve `b`
- If `b` is odd: `acc + a * b = (acc + a) + a * (b-1)` → add `a` to `acc`, decrement `b`

## Code

```scheme
(define (double x) (+ x x))
(define (halve x) (/ x 2))

(define (fast-mult a b)
  (define (iter acc a b)
    (cond ((= b 0) acc)
          ((even? b) (iter acc (double a) (halve b)))
          (else (iter (+ acc a) a (- b 1)))))
```

The process is iterative (tail-recursive) because all state is carried in the parameters `acc`, `a`, and `b`. It uses O(log b) steps since we halve `b` on even iterations.

## The GCD Procedure

```scheme
(define (gcd a b)
  (if (= b 0)
      a
      (gcd b (remainder a b))))
```

## Applicative-Order Evaluation (Normal Scheme)

Arguments are evaluated before being passed. Each call does 1 remainder:

```
(gcd 206 40)
(gcd 40 (remainder 206 40)) → (gcd 40 6)      ; 1 remainder
(gcd 6 (remainder 40 6))    → (gcd 6 4)       ; 1 remainder
(gcd 4 (remainder 6 4))     → (gcd 4 2)       ; 1 remainder
(gcd 2 (remainder 4 2))     → (gcd 2 0)       ; 1 remainder
→ 2
```

**Applicative-order: 4 remainder operations**

## Normal-Order Evaluation

Arguments are NOT evaluated until needed. They accumulate as expressions:

```
(gcd 206 40)
  b=40, not 0 →
(gcd 40 (r 206 40))
  b=(r 206 40), need to eval for test: =6, not 0 → [1 remainder]
```
